Control Scales

System Settings

Note that the system settings mostly control whether something can happen at all.
Settings in the evaluation can still be controlled on a per evaluation basis unless overridden here

Instructor Settings:

Instructors allowed to create evaluations
Instructors allowed to view results
Instructors allowed to email students
Instructors must use evaluations from above in the hierachy
Instructors may add this number of questions to evaluations from above in the hierarchy before the release date

Student Settings:

Students may leave questions unanswered
Students may modify their responses up to the due date
Students allowed to view results

Administrator Settings:

Admins below the hierarchy level of the owner may add this number of questions to the evaluation before the release date
Admins allowed to view instructor added question results
Only super admins may modify question text after eval start

General Settings:

Helpdesk email address (email messages appear to come from this email address by default)
Number of responses required before results can be viewed
Not Available allowed when creating questions
Maximum number of questions in a question block
  Template sharing and visibility setting
All questions default to course category only (instructor category not allowed)
Use stop date for evaluations
Use expert created templates
Use expert created questions
Use same view date for all users
(or use seperate dates for students, instructors, admins)
